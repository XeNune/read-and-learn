# 📖 Читай в оригинале - Платформа для обучения языкам

Интерактивная платформа для чтения текстов с поддержкой переводов, грамматических объяснений и многоязычного контента.

---

## 📚 Как добавить новый текст на платформу?

### Шаг 1️⃣: Подготовка текстового файла

1. Откройте папку `/public/texts/`
2. Создайте новый текстовый файл с названием, например: `text3.txt`
3. Скопируйте туда текст на нужном языке
4. Сохраните файл

**Пример содержимого файла `text3.txt`:**
```
The story begins in a small village where magic still exists. A young apprentice discovers an ancient book hidden in the library's basement.
```

---

### Шаг 2️⃣: Добавление информации о тексте в CSV

1. Откройте файл `/public/data/texts.csv`
2. Добавьте новую строку с информацией о тексте
3. Формат (разделитель - запятая):
   ```
   file,title,language,words
   text3.txt,Волшебная деревня,English,156
   ```

**Что означает каждое поле:**
- `file` - имя файла, который вы создали (должно совпадать!)
- `title` - красивое название для отображения на сайте
- `language` - язык оригинала
- `words` - приблизительное количество слов в тексте

**Текущие примеры:**
```
text1.txt,Старый дубовый ствол,English,172
text2.txt,Древняя библиотека,English,174
```

---

### Шаг 3️⃣: Создание словаря переводов

> **Что это?** JSON - это обычный текстовый формат для хранения данных. Выглядит красиво и структурированно.

1. Откройте файл `/public/data/translations.json`
2. Это файл со всеми переводами слов
3. Формат очень простой:

```json
{
  "word": "перевод",
  "another": "ещё один перевод",
  "tree": "дерево"
}
```

**Как это читать:**
- Слово слева (в кавычках) - оригинальное слово из текста
- Слово справа (тоже в кавычках) - его перевод
- Между ними двоеточие `:`
- После каждой пары запятая `,` (кроме последней!)

**Что делать:**
1. Откройте свой новый текст (text3.txt)
2. Найдите все слова, которые есть в нём
3. Добавьте их в `translations.json` в формате выше
4. Убедитесь, что последняя пара БЕЗ запятой

**Пример для текста про деревню:**
```json
{
  "story": "история",
  "begins": "начинается",
  "village": "деревня",
  "magic": "магия",
  "exists": "существует",
  "young": "молодой",
  "apprentice": "ученик",
  "discovers": "обнаруживает",
  "ancient": "древний",
  "book": "книга",
  "hidden": "скрытый",
  "library": "библиотека",
  "basement": "подвал"
}
```

> ⚠️ **Важно!** Каждое слово должно быть в нижнем регистре (маленькие буквы)!

---

### Шаг 4️⃣: Создание красивых переводов предложений

1. Откройте файл `/public/data/sentence-translations.json`
2. Это файл с полными переводами ПРЕДЛОЖЕНИЙ (не слов)
3. Здесь переводы звучат красиво и естественно

**Структура файла:**

```json
{
  "text3.txt": [
    {
      "english": "The story begins in a small village where magic still exists.",
      "russian": "История начинается в маленькой деревне, где магия всё ещё существует."
    },
    {
      "english": "A young apprentice discovers an ancient book hidden in the library's basement.",
      "russian": "Молодой ученик обнаруживает древнюю книгу, спрятанную в подвале библиотеки."
    }
  ]
}
```

**Что означает структура:**
- `"text3.txt":` - это ключ (название файла, к которому относятся переводы)
- `[ ]` - это массив (список) предложений
- `{ }` - это один объект (одно предложение + его перевод)
- `"english":` - оригинальное предложение (может быть любой язык!)
- `"russian":` - его красивый перевод (всегда на русский)

**Пошагово:**
1. Скопируйте структуру выше в `sentence-translations.json`
2. Замените `"text3.txt"` на имя вашего файла
3. Замените каждое предложение из вашего текста
4. Напишите красивый перевод каждого предложения
5. Убедитесь, что в JSON всё правильно (запятые, кавычки на месте)

> 💡 **Совет!** Используйте профессиональный переводчик (Google Translate, DeepL) как основу, а потом отредактируйте для естественного звучания.

---

### Шаг 5️⃣: Добавление грамматических правил (опционально)

> **Что это?** Полный перевод текста, который показывается при нажатии на кнопку "Перевести весь текст"

1. Откройте файл `/public/data/grammar.json`
2. Это файл с грамматическими объяснениями для слов
3. Когда пользователь кликает на слово и нажимает "Подробнее", показывается грамматика отсюда

**Структура файла:**

```json
{
  "articles": {
    "a": "Неопределённый артикль...",
    "the": "Определённый артикль..."
  },
  "nouns": {
    "tree": "Существительное, множественное число - trees",
    "book": "Существительное, множественное число - books"
  },
  "verbs": {
    "to_be": "Глагол быть/быть в глаголе быть..."
  }
}
```

**Что делать:**
1. Откройте свой текст и найдите сложные грамматические конструкции
2. Добавьте их в соответствующие категории в `grammar.json`
3. Напишите объяснение на русском

**Пример добавления:**
```json
{
  "verbs": {
    "discovered": "Глагол to discover в Past Simple (прошедшее время). Значение: обнаружить, найти. Образование: discover + ed"
  }
}
```

> ℹ️ **Совет:** Грамматика добавляется по мере надобности - это опционально. Если слова нет в grammar.json, то будет написано "Объяснение не добавлено"

---

### Шаг 6️⃣: Добавление полного перевода текста

> **Что это?** Это полный перевод всего текста целиком, который показывается при нажатии на "Перевести весь текст"

**Вариант 1 (рекомендуемый - просто создайте файл с переводом):**

1. Откройте папку `/public/texts/`
2. Создайте файл с полным переводом:
   - Если оригинал: `text3.txt`
   - То создайте: `text3.translated.txt`
3. Скопируйте в него полный перевод на русском

**Пример:**
```
/public/texts/
├── text1.txt                    ← оригинальный английский текст
├── text1.translated.txt         ← полный русский перевод
├── text2.txt                    ← оригинальный английский текст
├── text2.translated.txt         ← полный русский перевод
└── text3.txt                    ← ваш новый английский текст
└── text3.translated.txt         ← ваш новый русский перевод
```

**Как это работает:**
- Система автоматически найдёт файл с расширением `.translated.txt`
- Когда вы нажимаете "Перевести весь текст", компонент загружает этот файл
- Никаких правок в коде не нужно!

## 🌍 Добавление текста на ДРУГОМ языке (например, испанском)

Всё то же самое! Система автоматически определит язки и будет работать.

### Пример: Добавление испанского текста

**Шаг 1:** Создайте файл `/public/texts/spanish1.txt` со испанским текстом

**Шаг 2:** Добавьте в `texts.csv`:
```
spanish1.txt,Сказка о волшебстве,Spanish,189
```

**Шаг 3:** Переводы слов в `translations.json`:
```json
{
  "érase": "был",
  "una": "один",
  "vez": "раз",
  "aldea": "деревня",
  "mágica": "волшебная"
}
```

**Шаг 4:** Переводы предложений в `sentence-translations.json`:

⭐ **ВАЖНО:** Первый ключ - это язык оригинала, второй ключ - ВСЕГДА `"russian"`:

```json
{
  "spanish1.txt": [
    {
      "spanish": "Erase una vez una aldea mágica.",
      "russian": "Жила-была когда-то одна волшебная деревня."
    },
    {
      "spanish": "Los pobladores protegían sus tradiciones.",
      "russian": "Жители защищали свои традиции."
    }
  ]
}
```

**Система это поддерживает?** ✅ **ДА!** Компонент автоматически:
- Определит, что первый ключ - "spanish"
- Будет сравнивать испанский текст из UI с этим ключом
- Вернёт русский перевод из "russian"

**Можно использовать любые названия языков:**
- `"french"`, `"german"`, `"italian"`, `"portuguese"` - любой язык оригинала
- Второй ключ ВСЕГДА `"russian"` - целевой язык

**Примеры для других языков:**
```json
{
  "french_text.txt": [
    {
      "french": "Bonjour, comment allez-vous?",
      "russian": "Привет, как дела?"
    }
  ]
}
```

```json
{
  "german_text.txt": [
    {
      "german": "Guten Tag, wie geht es Ihnen?",
      "russian": "Добрый день, как ваши дела?"
    }
  ]
}
```

## 🐛 Частые ошибки и их решения

### ❌ "Текст не появляется на сайте"

**Решение:**
1. Проверьте, совпадают ли имена файлов в `texts.csv` и `/public/texts/`
2. Откройте браузер F12 (инструменты разработчика)
3. Перейдите на вкладку "Console" - там будут ошибки

### ❌ "Переводы не работают"

**Решение:**
1. Откройте `/public/data/translations.json`
2. Используйте онлайн-валидатор: https://jsonlint.com/
3. Скопируйте туда содержимое файла
4. Он покажет ошибку (не хватает запятой, неправильные кавычки и т.д.)

### ❌ "В переводах предложений нет результата"

**Решение:**
1. Проверьте, что имя ключа совпадает с файлом: `"text3.txt"` (точное совпадение)
2. Убедитесь, что предложения ТОЧНО совпадают (даже пробелы!)
3. Очистите кэш браузера: Ctrl+Shift+Delete

### ❌ "JSON не валидный"

**Как проверить JSON:**
1. Перейдите на https://jsonlint.com/
2. Скопируйте содержимое файла
3. Нажмите "Validate JSON"
4. Появится описание ошибки с номером строки

**Частые ошибки JSON:**
- Забыли запятую между элементами
- Использовали одинарные кавычки вместо двойных
- Забыли закрывающую скобку `}` или `]`

## � Как система работает с языками? (Техническое объяснение)

**Система универсальна и работает с ЛЮБЫМИ языками!**

### Как это работает внутри:

Когда вы добавляете текст на испанском:

```json
{
  "spanish_cat.txt": [
    {
      "spanish": "El gato negro dormía bajo el sol.",
      "russian": "Чёрный кот спал под солнцем."
    }
  ]
}
```

Компонент **автоматически:**

1. **Находит первый ключ** → `"spanish"` (это язык оригинала)
2. **Находит второй ключ** → `"russian"` (это язык целевой)
3. **При клике на слово берёт значение из первого ключа** → `"El gato negro dormía bajo el sol."`
4. **При совпадении возвращает значение из второго ключа** → `"Чёрный кот спал под солнцем."`

Это значит:
- ✅ Вы можете назвать первый ключ как угодно: `"spanish"`, `"spanish_es"`, `"español"`, `"es"` - любое название
- ✅ Второй ключ ДОЛЖЕН быть `"russian"` - это целевой язык
- ✅ Система сама поймёт, какой язык оригинала

### Пример с французским:
```json
{
  "french_tale.txt": [
    {
      "francais": "Bonjour, je m'appelle Pierre.",
      "russian": "Привет, меня зовут Пьер."
    }
  ]
}
```

Система поймёт:
- Первый ключ (`francais`) = французский текст
- Второй ключ (`russian`) = русский перевод

**Вывод:** Вы можете добавлять тексты на ЛЮБОМ языке, и система будет работать! 🌍

## ✨ Поддерживаемые языки

**Универсально работает с ЛЮБЫМ языком!** Примеры:

- **Английский:** `"english"` / `"russian"`
- **Испанский:** `"spanish"` или `"español"` / `"russian"`
- **Французский:** `"french"` или `"francais"` / `"russian"`
- **Немецкий:** `"german"` или `"deutsch"` / `"russian"`
- **Итальянский:** `"italian"` или `"italiano"` / `"russian"`
- **Португальский:** `"portuguese"` или `"português"` / `"russian"`
- **И любой другой язык!**

**Правило:** Первый ключ = любое название языка оригинала, второй ключ = ВСЕГДА `"russian"`

---

**Готово! Теперь вы можете добавлять тексты на любых языках! 🌍📚**

Если что-то не понятно, проверьте примеры выше или используйте JSON-валидатор!
